# Sync Progress to PR

Sync task progress to its associated Pull Request. Argument: task-id.

## Execution Flow

### 1. Verify Task Exists

Search for the task in priority order:
- `.agent-workspace/active/{task-id}/task.md`
- `.agent-workspace/completed/{task-id}/task.md`
- `.agent-workspace/blocked/{task-id}/task.md`

Note: `{task-id}` format is `TASK-{yyyyMMdd-HHmmss}`, e.g. `TASK-20260306-143022`

### 2. Read Task Information

From task.md, extract:
- `pr_number` (required - if missing, prompt user)
- Task title, description, status
- `current_step`, `created_at`, `updated_at`
- `issue_number` (if applicable)

### 3. Read Context Files

Check and read (if they exist):
- `analysis.md` - Requirement analysis
- `plan.md` - Technical plan
- `implementation.md` - Implementation report
- `review.md` - Review report

### 4. Generate Progress Summary

Generate a clear progress summary oriented toward **code reviewers**:

```markdown
## Development Progress Update

**Task ID**: {task-id}
**Updated**: {current time}
**Status**: {status description}

### Completed Steps

- [x] Requirement Analysis - {completion time}
  - {1-2 key takeaways}
- [x] Technical Design - {completion time}
  - {Key decisions}
- [x] Implementation - {completion time}
  - Files modified: {count}
  - Tests added: {count}
- [ ] Code Review (in progress)
- [ ] Final Merge

### Current Progress

{Detailed description of current step}

### Next Steps

{What needs to happen next}

### Technical Highlights

{Key technical decisions and implementation details for reviewers}

### Related Documents

- Task: `.agent-workspace/{status}/{task-id}/task.md`
- Analysis: `.agent-workspace/{status}/{task-id}/analysis.md`
- Plan: `.agent-workspace/{status}/{task-id}/plan.md`
- Implementation: `.agent-workspace/{status}/{task-id}/implementation.md`

---
*Auto-generated by AI - [Task Management](.agents/README.md)*
```

**Summary principles**:
- **Reviewer-oriented**: Highlight technical decisions and implementation details
- **Concise**: Extract only key points per phase
- **Logical flow**: Present progress chronologically
- **Review-friendly**: Explain why key changes were made

### 5. Post to PR

```bash
gh pr comment {pr-number} --body "$(cat <<'EOF'
{generated summary}
EOF
)"
```

### 6. Update Task Status

Get the current time:

```bash
date "+%Y-%m-%d %H:%M:%S"
```

Add or update `last_synced_to_pr_at` field in task.md to `{current time}`.
- **Append** to `## Activity Log` (do NOT overwrite previous entries):
  ```
  - {yyyy-MM-dd HH:mm:ss} — **Sync to PR** by {agent} — Progress synced to PR #{pr-number}
  ```

### 7. Inform User

```
Progress synced to PR #{pr-number}.

Synced content:
- Completed steps: {count}
- Current status: {status}
- Next step: {description}

View: https://github.com/{owner}/{repo}/pull/{pr-number}
```

## Notes

1. **PR number required**: Task must have `pr_number` in task.md. If missing, prompt user.
2. **Audience**: The `sync-pr` skill is for code reviewers; the `sync-issue` skill is for stakeholders. Different focus.
3. **Timing**: Sync after completing major phases, not after every small change.
4. **Avoid spam**: Don't sync too frequently - consolidate updates.

## Error Handling

- Task not found: Prompt "Task {task-id} not found"
- Missing PR number: Prompt "Task has no pr_number field"
- PR not found: Prompt "PR #{number} not found"
- PR closed: Prompt "PR #{number} is already closed"
- gh auth failure: Prompt "Please check GitHub CLI authentication"
